# TODO 골든패스 — 의사결정 기록

> 이 파일은 PO와 개발자가 3-layer 하네스를 함께 사용하며 내린 모든 결정을 추적한다.
> 자동으로 기록된 항목과 수동 입력 항목이 섞여 있다.

---

## D-001 백엔드 기술 스택
- **결정**: Node.js + Express + TypeScript (Java/Spring Boot 아님)
- **근거**: 골든패스 시연 목적 — 빌드/실행 속도, 환경 세팅 단순성 우선. 하네스 원칙은 스택 무관하게 동일하게 적용됨.
- **결정자**: 개발 리드 (Claude Code)
- **날짜**: 2026-06-22

## D-002 데이터베이스
- **결정**: SQLite (개발/데모) — 운영시 PostgreSQL 교체
- **근거**: 골든패스는 로컬 단독 실행이 목표. DB 설치 없이 파일 기반으로 동작. tech-stack.md에 기재.
- **결정자**: 개발 리드
- **날짜**: 2026-06-22

## D-003 인증 방식
- **결정**: JWT (Access Token 메모리 저장, Refresh Token httpOnly 쿠키)
- **근거**: SSO가 없는 순수 데모 환경. SPEC-000 Phase 0 모드 B(직접 구현).
- **결정자**: 개발 리드
- **날짜**: 2026-06-22

## D-004 디자인 페이지 추가 (DP-LOGIN)
- **결정**: example 프로젝트 DP-MAIN/DP-POPUP 외에 DP-LOGIN(인증 전용 페이지) 추가
- **근거**: 로그인 화면은 Header/NavMenu가 없고 중앙 정렬 카드 레이아웃이 별개 archetype임
- **결정자**: 개발 리드
- **날짜**: 2026-06-22

## D-005 TODO 진행상황 팝업 데이터 모델
- **결정**: Todo 1개에 Progress 기록 N개 (히스토리 방식)
- **근거**: PO 요구사항 — "클릭하면 나오는 progress 관리 팝업". 단순 상태 변경이 아닌 히스토리 추적.
- **결정자**: PO + 개발 리드 합의
- **날짜**: 2026-06-22

## D-006 다크모드
- **결정**: 기본 다크모드 활성화 (시스템 prefers-color-scheme 연동 + 수동 토글)
- **근거**: "모던한 UI" 요구사항. shadcn/ui 다크모드 지원 내장.
- **결정자**: 개발 리드
- **날짜**: 2026-06-22

## D-007 Phase 0 공통 기능 전달 모드
